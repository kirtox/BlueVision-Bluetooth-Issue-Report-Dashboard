# Alembic 資料庫遷移設定指南

## 概述
Alembic 是 SQLAlchemy 的資料庫遷移工具，用於管理資料庫結構的版本控制。

## 已完成的設定

### 1. 依賴項目
已在 requirements.txt 中加入 alembic

### 2. 配置文件
- alembic.ini: Alembic 主配置文件
- alembic/env.py: 環境配置，已設定導入 app.models
- alembic/script.py.mako: 遷移腳本模板
- alembic/versions/: 遷移腳本存放目錄

### 3. 管理腳本
- manage_migrations.py: 遷移管理腳本
- init_alembic.py: 初始化腳本（備用）

## 執行步驟

### 步驟 1: 重新建置容器
```bash
# 重新建置容器（安裝 alembic）
podman-compose -f podman-compose.prod.yml down
podman-compose -f podman-compose.prod.yml up --build -d
```

### 步驟 2: 進入容器
```bash
podman exec -it <backend_container_name> bash
```

### 步驟 3: 執行遷移
```bash
# 在容器內執行以下命令：

# 1. 創建初始遷移（基於現有資料表）
python manage_migrations.py init

# 2. 升級資料庫
python manage_migrations.py upgrade

# 3. 檢查當前版本
python manage_migrations.py current
```

### 步驟 4: 如果需要更新 user 表
```bash
# 創建 user 表更新遷移
python manage_migrations.py user

# 升級資料庫
python manage_migrations.py upgrade
```

### 步驟 5: 驗證結果
```bash
# 查看遷移歷史
python manage_migrations.py history
```

## 管理腳本使用方法

```bash
python manage_migrations.py init          # 創建初始遷移
python manage_migrations.py user          # 創建 user 表更新遷移
python manage_migrations.py upgrade       # 升級資料庫
python manage_migrations.py current       # 顯示當前版本
python manage_migrations.py history       # 顯示遷移歷史
```

## 常用 Alembic 命令

### 創建遷移
```bash
# 自動檢測模型變更並創建遷移
alembic revision --autogenerate -m "描述變更內容"

# 手動創建空白遷移
alembic revision -m "描述變更內容"
```

### 執行遷移
```bash
# 升級到最新版本
alembic upgrade head

# 升級到特定版本
alembic upgrade <revision_id>

# 回滾一個版本
alembic downgrade -1

# 回滾到特定版本
alembic downgrade <revision_id>
```

### 查看狀態
```bash
# 查看當前版本
alembic current

# 查看遷移歷史
alembic history

# 查看詳細歷史
alembic history --verbose
```

```
sqlalchemy.url = postgresql://admin:password@127.0.0.1:5433/btird
```

## User 表結構更新

更新後的 User 模型包含：
- id: 主鍵
- username: 用戶名（唯一）
- hashed_password: 加密後的密碼
- role: 權限等級（"Administrator" 或 "User"，預設 "User"）
- created_at: 創建時間（預設當前時間）
- is_active: 是否啟用（"Y" 或 "N"，預設 "Y"）

## Alembic 的優點

1. **版本控制**: 每個資料庫變更都有版本記錄
2. **可回滾**: 可以回滾到之前的版本
3. **自動生成**: 可以自動檢測模型變更並生成遷移腳本
4. **團隊協作**: 遷移腳本可以版本控制，團隊成員可以同步資料庫結構
5. **安全性**: 可以在執行前預覽 SQL 語句

## 注意事項

1. 每次修改模型後，都應該創建新的遷移
2. 遷移腳本應該加入版本控制系統
3. 在生產環境執行遷移前，建議先在測試環境驗證
4. 重要資料變更前，建議先備份資料庫

## 故障排除

### 如果遇到 "Target database is not up to date" 錯誤：
```bash
# 標記當前資料庫狀態為最新
alembic stamp head
```

### 如果需要重置遷移歷史：
```bash
# 刪除 alembic_version 表
# 重新創建初始遷移
python manage_migrations.py init
```

### 如果遇到模型導入錯誤：
檢查 alembic/env.py 中的模型導入路徑是否正確。
