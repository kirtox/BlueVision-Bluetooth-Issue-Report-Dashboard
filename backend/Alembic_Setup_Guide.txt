# Alembic Database Migration Setup Guide

## Overview
Alembic is SQLAlchemy's database migration tool for managing database schema version control.

## Completed Setup

### 1. Dependencies
Added alembic to requirements.txt

### 2. Configuration Files
- alembic.ini: Alembic main configuration file
- alembic/env.py: Environment configuration, set to import app.models
- alembic/script.py.mako: Migration script template
- alembic/versions/: Migration script storage directory

### 3. Management Scripts
- manage_migrations.py: Migration management script
- init_alembic.py: Initialization script (backup)

## Execution Steps

### Step 1: Rebuild Container
```bash
# Rebuild container (install alembic)
podman-compose -f podman-compose.prod.yml down
podman-compose -f podman-compose.prod.yml up --build -d
```

### Step 2: Enter Container
```bash
podman exec -it <backend_container_name> bash
```

### Step 3: Execute Migration
```bash
# Execute the following commands inside container:

# 1. Create initial migration (based on existing tables)
python manage_migrations.py init

# 2. Upgrade database
python manage_migrations.py upgrade

# 3. Check current version
python manage_migrations.py current
```

### Step 4: If user table update is needed
```bash
# Create user table update migration
python manage_migrations.py user

# Upgrade database
python manage_migrations.py upgrade
```

### Step 5: Verify Results
```bash
# View migration history
python manage_migrations.py history
```

## Management Script Usage

```bash
python manage_migrations.py init          # Create initial migration
python manage_migrations.py user          # Create user table update migration
python manage_migrations.py upgrade       # Upgrade database
python manage_migrations.py current       # Show current version
python manage_migrations.py history       # Show migration history
```

## Common Alembic Commands

### Creating Migrations
```bash
# Auto-detect model changes and create migration
alembic revision --autogenerate -m "Describe changes"

# Manually create blank migration
alembic revision -m "Describe changes"
```

### Executing Migrations
```bash
# Upgrade to latest version
alembic upgrade head

# Upgrade to specific version
alembic upgrade <revision_id>

# Rollback one version
alembic downgrade -1

# Rollback to specific version
alembic downgrade <revision_id>
```

### Viewing Status
```bash
# View current version
alembic current

# View migration history
alembic history

# View detailed history
alembic history --verbose
```

## User Table Structure Update

Updated User model contains:
- id: Primary key
- username: Username (unique)
- hashed_password: Encrypted password
- role: Permission level ("Administrator" or "User", default "User")
- created_at: Creation time (default current time)
- is_active: Whether enabled ("Y" or "N", default "Y")

## Alembic Advantages

1. **Version Control**: Each database change has version record
2. **Rollback Capability**: Can rollback to previous versions
3. **Auto-generation**: Can automatically detect model changes and generate migration scripts
4. **Team Collaboration**: Migration scripts can be version controlled, team members can sync database structure
5. **Security**: Can preview SQL statements before execution

## Notes

1. Should create new migration after each model modification
2. Migration scripts should be added to version control system
3. Recommend testing in test environment before executing migrations in production
4. Recommend backing up database before important data changes

## Troubleshooting

### If encountering "Target database is not up to date" error:
```bash
# Mark current database state as latest
alembic stamp head
```

### If need to reset migration history:
```bash
# Delete alembic_version table
# Recreate initial migration
python manage_migrations.py init
```

### If encountering model import errors:
Check if the model import path in alembic/env.py is correct.
