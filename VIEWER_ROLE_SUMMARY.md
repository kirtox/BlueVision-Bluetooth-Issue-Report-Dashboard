# Auditor 角色權限總結

## 新增的 Auditor 角色特點

Auditor 角色是一個介於 User 和 Administrator 之間的權限層級，主要特點如下：

### ✅ Auditor 可以做的事情：
- **創建報告** - 類似 User 角色
- **編輯所有報告** - 不限於自己創建的報告（這是與 User 的主要區別）
- **刪除所有報告** - 不限於自己創建的報告
- **匯出報告** - 可以匯出系統中的報告
- **查看所有報告** - 可以瀏覽系統中的所有報告
- **編輯個人資料** - 可以修改自己的個人資料
- **查看平台狀態** - 可以查看平台相關資訊

### ❌ Auditor 不能做的事情：
- **管理用戶** - 無法創建、編輯或刪除其他用戶
- **查看 API 日誌** - 無法查看系統的 API 存取日誌
- **管理平台** - 無法管理平台設定

| 查看平台 | ✓ | ✓ | ✓ | ✓ |

## 使用場景

Auditor 角色適合以下使用場景：

1. **報告審計人員** - 需要能夠審查和修正所有報告，確保資料準確性
2. **品質控制專員** - 負責檢查報告品質，修正錯誤和不一致之處
3. **合規檢查員** - 需要審核報告內容是否符合標準和規範
4. **資料驗證員** - 負責驗證和清理報告資料
