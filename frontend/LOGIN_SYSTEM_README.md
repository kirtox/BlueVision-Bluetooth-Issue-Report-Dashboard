# 登入系統使用說明

## 概述

基於 Header.tsx 中的 Notifications 組件設計，我們建立了一個完整的用戶認證系統。

## 主要功能

### 1. 用戶認證 Context (`AuthContext.tsx`)
- 提供全局的用戶狀態管理
- 支持登入、登出功能
- 自動檢查本地存儲的認證狀態
- 提供 loading 狀態管理

### 2. 用戶菜單組件 (`UserMenu.tsx`)
- 基於原有 Notifications 組件的設計風格
- 未登入時顯示登入/註冊按鈕
- 已登入時顯示用戶頭像和下拉菜單
- 包含用戶資訊、設定選項和登出功能

### 3. 登入頁面 (`SignIn.tsx`)
- 中文化的登入表單
- 表單驗證和錯誤處理
- 自動重定向到主頁面
- Loading 狀態顯示

### 4. 註冊頁面 (`SignUp.tsx`)
- 中文化的註冊表單
- 密碼確認驗證
- 服務條款同意檢查
- 自動登入功能

### 5. 路由保護 (`ProtectedRoute.tsx`)
- 保護需要認證的頁面
- 自動重定向未登入用戶到登入頁面
- Loading 狀態處理

## 使用方式

### 1. 訪問登入頁面
- 直接訪問 `/signin` 進入登入頁面
- 或點擊 Header 中的「登入」按鈕

### 2. 註冊新帳戶
- 訪問 `/signup` 進入註冊頁面
- 或從登入頁面點擊「建立新帳戶」

### 3. 測試認證系統
- 登入後訪問 `/pages/auth-test` 查看認證狀態
- 可以查看當前用戶資訊和系統狀態

### 4. 登出
- 點擊用戶頭像下拉菜單中的「登出」選項

## 雙模式認證系統

### 🔄 模式切換
系統支持兩種認證模式，可在登入/註冊頁面頂部切換：

1. **模擬模式**（預設）
   - 任何用戶名和密碼都可以登入
   - 適合開發和測試
   - 不需要後端服務

2. **API 模式**
   - 連接到真實的後端 API
   - 需要有效的用戶帳戶
   - 支持真實的註冊和登入

### 🔧 後端 API 端點

- `POST /login` - 用戶登入
- `POST /users` - 用戶註冊
- `GET /verify-token` - Token 驗證

### 🛠️ 技術實現

- **密碼安全**: 使用 bcrypt 哈希
- **Token 管理**: JWT Token 認證
- **狀態持久化**: localStorage 存儲
- **錯誤處理**: 完整的錯誤提示

## 注意事項

- 預設使用模擬模式，適合快速開發測試
- API 模式需要後端服務運行在 `VITE_API_BASE_URL` 配置的地址
- 用戶資料存儲在瀏覽器的 localStorage 中
- 生產環境建議使用 API 模式並配置適當的安全措施
