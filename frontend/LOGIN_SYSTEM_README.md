# 登入系統使用說明

## 概述

基於 Header.tsx 中的 Notifications 組件設計，我們建立了一個完整的用戶認證系統。

## 主要功能

### 1. 用戶認證 Context (`AuthContext.tsx`)
- 提供全局的用戶狀態管理
- 支持登入、登出功能
- 自動檢查本地存儲的認證狀態
- 提供 loading 狀態管理

### 2. 用戶菜單組件 (`UserMenu.tsx`)
- 基於原有 Notifications 組件的設計風格
- 未登入時顯示登入/註冊按鈕
- 已登入時顯示用戶頭像和下拉菜單
- 包含用戶資訊、設定選項和登出功能

### 3. 登入頁面 (`SignIn.tsx`)
- 中文化的登入表單
- 表單驗證和錯誤處理
- 自動重定向到主頁面
- Loading 狀態顯示

### 4. 註冊頁面 (`SignUp.tsx`)
- 中文化的註冊表單
- 密碼確認驗證
- 服務條款同意檢查
- 自動登入功能

### 5. 路由保護 (`ProtectedRoute.tsx`)
- 保護需要認證的頁面
- 自動重定向未登入用戶到登入頁面
- Loading 狀態處理

## 使用方式

### 1. 訪問登入頁面
- 直接訪問 `/signin` 進入登入頁面
- 或點擊 Header 中的「登入」按鈕

### 2. 註冊新帳戶
- 訪問 `/signup` 進入註冊頁面
- 或從登入頁面點擊「建立新帳戶」

### 3. 測試認證系統
- 登入後訪問 `/pages/auth-test` 查看認證狀態
- 可以查看當前用戶資訊和系統狀態

### 4. 登出
- 點擊用戶頭像下拉菜單中的「登出」選項

## 技術特點

1. **基於現有設計**: 完全基於 Header.tsx 中 Notifications 組件的設計風格
2. **響應式設計**: 支持桌面和移動設備
3. **中文化**: 所有界面文字都已中文化
4. **狀態管理**: 使用 React Context 進行全局狀態管理
5. **本地存儲**: 支持記住登入狀態
6. **路由保護**: 自動保護需要認證的頁面

- 生產環境需要連接實際的後端認證服務
